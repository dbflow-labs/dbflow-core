# DBFlow v1.1 Database Schema Reference

All v1.1 schema changes are **additive**. Historical rows are never fabricated.

## SLA (Stage 1.1-C)

### `dbflow_workflow_tasks` (new columns)

| Column | Type | Description |
| --- | --- | --- |
| `overdue_at` | timestamp nullable | Set when overdue processing marks the task |
| `sla_policy_snapshot` | json nullable | Frozen SLA policy at task creation |
| `sla_policy_source` | string nullable | `v1.1_sla` when v1.1 SLA applies; null for legacy timeout only |

### `dbflow_workflow_sla_events`

Event ledger for reminders, overdue markers, and escalations.

| Column | Description |
| --- | --- |
| `workflow_task_id`, `workflow_instance_id`, `node_key` | Foreign keys and node identity |
| `event_type` | `reminder`, `overdue`, `escalation` |
| `sequence` | Ordering within task |
| `scheduled_at` | When the event becomes due |
| `status` | `pending`, `processing`, `completed`, `failed`, `cancelled` |
| `idempotency_key` | Unique business key |
| `attempts`, `max_attempts`, `next_attempt_at` | Retry state |
| `processing_started_at`, `processed_at`, `failed_at`, `cancelled_at` | Lifecycle timestamps |
| `policy_snapshot`, `result_metadata`, `last_error` | Safe audit data |

**Unique:** `idempotency_key`

## Reliable Actions (Stage 1.1-D)

### `dbflow_workflow_action_executions`

See `docs/architecture/reliable-action-runtime.md` for the execution ledger schema.

### `dbflow_workflow_action_attempts`

Per-attempt evidence linked to executions.

**No rows** are created for `legacy_sync` actions.
