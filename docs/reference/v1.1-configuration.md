# DBFlow v1.1 Configuration Reference

Configuration lives in `config/dbflow.php`. Environment variables are listed where applicable.

## SLA (`dbflow.sla`)

| Key | Env | Default | Description |
| --- | --- | --- | --- |
| `min_duration_seconds` | `DBFLOW_SLA_MIN_DURATION_SECONDS` | `60` | Minimum SLA duration |
| `max_duration_seconds` | `DBFLOW_SLA_MAX_DURATION_SECONDS` | `31536000` | Maximum SLA duration |
| `max_reminder_count` | `DBFLOW_SLA_MAX_REMINDER_COUNT` | `10` | Max reminders per approval node |
| `max_attempts` | `DBFLOW_SLA_MAX_ATTEMPTS` | `3` | Default max handler attempts |
| `max_backoff_seconds` | `DBFLOW_SLA_MAX_BACKOFF_SECONDS` | `86400` | Cap for retry backoff |
| `default_backoff_seconds` | — | `[60, 300, 900]` | Default backoff schedule |
| `dispatch_batch_size` | `DBFLOW_SLA_DISPATCH_BATCH_SIZE` | `100` | `dbflow:sla-dispatch` batch limit |
| `recovery_batch_size` | `DBFLOW_SLA_RECOVERY_BATCH_SIZE` | `100` | `dbflow:sla-recover` batch limit |
| `stale_processing_threshold_seconds` | `DBFLOW_SLA_STALE_THRESHOLD_SECONDS` | `900` | Stale processing detection |
| `default_notification_channel` | `DBFLOW_SLA_DEFAULT_CHANNEL` | `event` | Default reminder channel |
| `allowed_notification_channels` | — | `['event']` | Registered channels |
| `require_registered_escalation_resolver` | `DBFLOW_SLA_REQUIRE_REGISTERED_ESCALATION_RESOLVER` | `true` | Escalation target validation |
| `max_error_length` | `DBFLOW_SLA_MAX_ERROR_LENGTH` | `1000` | Sanitized error cap |

## Reliable Actions (`dbflow.actions`)

| Key | Env | Default | Description |
| --- | --- | --- | --- |
| `max_attempts` | `DBFLOW_ACTIONS_MAX_ATTEMPTS` | `3` | Default max handler attempts |
| `default_backoff_seconds` | `DBFLOW_ACTIONS_DEFAULT_BACKOFF_SECONDS` | `60` | Retry backoff |
| `max_backoff_seconds` | `DBFLOW_ACTIONS_MAX_BACKOFF_SECONDS` | `86400` | Backoff cap |
| `dispatch_batch_size` | `DBFLOW_ACTIONS_DISPATCH_BATCH_SIZE` | `100` | `dbflow:actions-dispatch` batch |
| `recovery_batch_size` | `DBFLOW_ACTIONS_RECOVERY_BATCH_SIZE` | `100` | `dbflow:actions-recover` batch |
| `stale_processing_threshold_seconds` | `DBFLOW_ACTIONS_STALE_THRESHOLD_SECONDS` | `900` | Stale running detection |
| `max_error_length` | `DBFLOW_ACTIONS_MAX_ERROR_LENGTH` | `1000` | Sanitized error cap |

## Outbound Webhooks (`dbflow.webhook`)

| Key | Env | Default | Description |
| --- | --- | --- | --- |
| `timeout_seconds` | `DBFLOW_WEBHOOK_TIMEOUT_SECONDS` | `30` | HTTP timeout |
| `max_request_body_length` | `DBFLOW_WEBHOOK_MAX_REQUEST_BODY_LENGTH` | `65536` | Request body size limit |
| `max_response_body_length` | `DBFLOW_WEBHOOK_MAX_RESPONSE_BODY_LENGTH` | `4096` | Response truncation |
| `max_header_count` | `DBFLOW_WEBHOOK_MAX_HEADER_COUNT` | `32` | Max custom headers |
| `max_header_value_length` | `DBFLOW_WEBHOOK_MAX_HEADER_VALUE_LENGTH` | `8192` | Max header value length |
| `deny_private_ips` | `DBFLOW_WEBHOOK_DENY_PRIVATE_IPS` | `true` | SSRF default-deny |
| `require_https` | `DBFLOW_WEBHOOK_REQUIRE_HTTPS` | `false` | Require HTTPS URLs |
| `follow_redirects` | `DBFLOW_WEBHOOK_FOLLOW_REDIRECTS` | `false` | Redirects disabled by default |
| `max_redirects` | `DBFLOW_WEBHOOK_MAX_REDIRECTS` | `3` | Redirect hop limit |
| `host_allowlist` | `DBFLOW_WEBHOOK_HOST_ALLOWLIST` | `[]` | Optional host allowlist (comma-separated) |
| `allowed_schemes` | — | `['https','http']` | Allowed URL schemes |
| `idempotency_header` | — | `X-DBFlow-Idempotency-Key` | Stable transport header |
| `timestamp_header` | — | `X-DBFlow-Timestamp` | Signing timestamp header |
| `signature_header` | — | `X-DBFlow-Signature` | HMAC signature header |
| `disallowed_headers` | — | see config | Rejected custom headers |
| `redacted_header_keys` | — | see config | Header redaction list |

See `docs/architecture/outbound-webhook-security.md` for signing and redirect algorithms.

## Legacy Timeout

v1.0 approval `config.timeout.due_in` and `reject_end` remain unchanged. Tasks with `sla_policy_source = v1.1_sla` are excluded from `dbflow:process-timeouts`.
