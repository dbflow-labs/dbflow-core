# DBFlow v1.1 Events Reference

This document catalogs Laravel events introduced or extended in DBFlow v1.1. Payloads must never include plaintext secrets.

## SLA Events (Stage 1.1-C)

| Event | When dispatched | Key payload fields |
| --- | --- | --- |
| `TaskSlaScheduled` | SLA events materialized for a task | `task`, `instance`, snapshot metadata |
| `TaskReminderDue` | Reminder event claimed for processing | `event`, `task`, `instance` |
| `TaskReminderDispatched` | Reminder notification succeeded (event channel) | `event`, `task`, `instance`, `channel` |
| `TaskReminderFailed` | Reminder handler failed | `event`, `task`, `instance`, `error`, `retry` |
| `TaskBecameOverdue` | Task marked overdue (first time) | `task`, `instance`, `due_at`, `sla_event_id` |
| `TaskEscalationDue` | Escalation event claimed | `event`, `task`, `instance` |
| `TaskEscalated` | Escalation handler succeeded | `event`, `task`, `instance`, escalation metadata |
| `TaskEscalationFailed` | Escalation handler failed | `event`, `task`, `instance`, `error`, `retry` |
| `SlaEventRecovered` | Stale SLA event returned to pending | `event`, `attempts`, `idempotency_key` |
| `SlaEventCancelled` | SLA events cancelled (task terminal, etc.) | task/instance context |

## Workflow Log Parity

SLA runtime also writes `WorkflowLog` entries:

- `task_became_overdue`
- `task_sla_reminder_dispatched`
- `task_sla_escalated`

## Reliable Action Events (Stage 1.1-D)

| Event | When dispatched |
| --- | --- |
| `ActionExecutionQueued` | Ledger row created |
| `ActionExecutionStarted` | Claimed for processing |
| `ActionExecutionSucceeded` | Handler succeeded |
| `ActionExecutionFailed` | Permanent handler failure |
| `ActionExecutionExhausted` | Retries exhausted |
| `ActionExecutionRetryScheduled` | Retryable failure re-queued |
| `ActionExecutionManuallyRetried` | Operator retry |
| `ActionExecutionSkipped` | Manual skip |
| `ActionExecutionCancelled` | Cancelled (workflow cancel, etc.) |
| `ActionExecutionRecovered` | Stale `running` returned to `queued` |
| `WebhookRequestDispatched` | Outbound request sent (redacted metadata) |
| `WebhookResponseReceived` | Successful HTTP response |
| `WebhookRequestFailed` | HTTP/transport failure |
| `WebhookSecurityPolicyRejected` | SSRF/template policy rejection |

See `docs/architecture/reliable-action-runtime.md` for workflow log parity (`action_execution_*` events).

## v1.0 Events (unchanged)

Legacy synchronous actions continue to dispatch `ActionFailed` on handler exceptions. Successful legacy actions log `action_executed` without additional Laravel events.
